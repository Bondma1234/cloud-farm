## 1. 项目概述

### 1.1 项目名称
云上田园 · 河南云农场

### 1.2 使命
**让每个城市家庭都有一块"看得见的安心农田"。**

### 1.3 核心模式
1. **认养制**：在河南（建议周口/驻马店）租赁土地 50 亩，切分为 10–20 ㎡ 标准模块，通过小程序出售「年度认养权」。
2. **可参与**：用户可在小程序内选地、选作物、发送"浇水/施肥/除草"指令（由场内农技员执行并回传照片）。
3. **可观看**：每个认养田块部署太阳能 4G 摄像头，24 小时慢直播，可远程云台控制（进阶版）。
4. **可收获**：作物成熟后按约定批次冷链寄送到用户手上，每户保底 10–20 斤。
5. **可分享**：用户把生长报告、收获照片分享到社交平台，形成二次传播。
6. **可带货**：自营地同步种植热门作物（水果玉米、贝贝南瓜、蜜薯等），每日抖音/视频号直播销售。

### 1.4 三年战略分期
- **第一年（2026）**：单基地模式跑通，建立 C 端（小程序 + Web 前台）+ Web 运营后台，完成首个 50 亩基地闭环。
- **第二年（2027）**：扩展 SKU 与增值服务（亲子研学、农场游学、企业团建），上线直播独立营收模块。
- **第三年（2028）**：复制一个华南基地，上线"多基地"版本，输出 SaaS 赋能给其他小型农场主。

## 2. 市场分析

### 2.1 用户痛点（需求侧）
| 痛点 | 表现 | 我们的回应 |
|---|---|---|
| 食品安全焦虑 | 农药残留、来源不明 | 全程直播 + 种植日志，可追溯 |
| 亲子教育刚需 | 城市儿童缺乏自然课 | 亲子版套餐 + 线上自然课 + 孩子名字立牌 |
| 陪伴式情感消费 | 云吸猫、云旅游流行 | 24h 慢直播作物，带"主角感" |
| CSA 参与感弱 | 传统配送单向、无互动 | 指令交互 + 生长报告 + 命名权 |
| 企业福利同质化 | 月饼/大米/卡券千篇一律 | 企业田 + 员工远程观看 + 定制包装 |

### 2.2 市场规模测算（TAM / SAM / SOM）
- **TAM**：一线城市常住人口约 8000 万，家庭数约 2600 万。
- **SAM**：对"食品安全+自然教育"付费意愿较强的中高收入家庭，保守估 260 万（占比 10%）。
- **SOM（可获得市场）**：首年渗透 0.03% ≈ 800 户；三年复合渗透 0.2% ≈ 5000 户。
- **客单价区间**：499 – 999 元/年（认养），100 – 300 元/单（直播电商），3000 – 20000 元/年（企业端）。

### 2.3 目标用户画像
**核心人群 A – "自然教育妈妈"**
- 年龄：30–42；职住：一线城市；年收入：家庭 40 万+
- 典型行为：知乎/小红书"有机""宝宝辅食"重度读者；给孩子报自然课、博物馆亲子营
- 付费意愿：亲子版 699 元/年，次年续费靠"孩子认出自己的田"

**核心人群 B – "松弛感白领"**
- 年龄：25–35；单身或丁克；消费力强
- 典型行为：买多肉、养猫、订花、追综艺
- 付费意愿：基础版 499 元/年，看中"情绪价值 + 社交货币"

**次核心人群 C – "企业行政/HR"**
- 诉求：低单价高触达的员工福利、ESG 故事点
- 付费意愿：单公司 5000–20000 元/年

### 2.4 竞品对比
| 对手 | 模式 | 弱点 | 我们的差异化 |
|---|---|---|---|
| 认养一头牛 | 认养奶牛，重供应链 | 仅交付产品，无参与感 | 我们交付"过程" |
| 本来生活/盒马 | 有机农产品零售 | 标准货架逻辑 | 我们是"长期关系" |
| 传统 CSA 农场 | 周配送礼盒 | 地方性、无 App | 我们是全国可远程 |
| 盆栽类 App（啫喱、江南百景图等） | 虚拟种菜 | 收获不真实 | 我们"虚实结合" |

## 3. 产品与服务设计

### 3.1 认养套餐体系（核心产品）

| 版本 | 面积 | 年费 | 核心权益 |
|---|---|---|---|
| **基础版** | 10 ㎡ | 499 元 | 1 种作物，保底收获 10 斤，包邮 2 次，共享摄像头轮播，每月生长报告 |
| **进阶版** | 15 ㎡ | 799 元 | 2 种作物，保底 20 斤，包邮 3 次，专属可遥控摄像头，优先场景选择权 |
| **亲子版** | 10 ㎡ | 699 元 | 基础权益 + 3 节线上自然教育课 + 实体种植工具包 + 孩子名字立牌 + 生长纪念相册 |
| **企业版** | 50 ㎡ 起 | 5000 元起/年 | 企业 Logo 立牌、员工分账号（最多 50 人观看）、定制包装、年度 ESG 报告 |

**作物策略**：
- 优先：耐储运 + 高观赏性 —— 红薯、胡萝卜、土豆、南瓜、草莓、小番茄、香椿、大蒜
- 避开：易腐叶菜（仅秋冬限量）
- 深加工：萝卜干、豆角干、辣椒酱、芝麻盐（解决物流与损耗）

### 3.2 互动玩法
- **指令系统**：浇水、施肥、除草、捉虫、拍照特写（有冷却时间，每周限额，防刷屏）
- **成长里程碑**：发芽、开花、结果、收获 4 节点自动推送，附短视频
- **命名权**：给田块起名 / 给作物起名（亲子版可印在立牌上）
- **土地流转**：用户可把地块转赠给朋友，促裂变
- **年度回忆录**：年底自动生成本季生长时间轴 + 短视频，方便用户转发

### 3.3 直播电商（辅助产品）
- **自营地 20–30 亩**：种水果玉米、贝贝南瓜、蜜薯等
- **直播频道**：每天 4–6 小时，固定栏目
  - 周一"今日田间巡视"
  - 周四"挖红薯大赛"
  - 周六"摘草莓直播"
- **SKU**：9.9 元尝鲜装（引流）、99 元月卡箱、999 元年卡（12 次配送）

### 3.4 增值服务（长尾）
- 生长报告纸质相册（+88 元/季）
- 特殊种子/稀缺品种（+50–200 元）
- 农场实地研学（周末 1 日游，含车费 580 元/人）
- 婚礼/生日田（纪念日种植服务，1500–3000 元）

> **C 端双端说明**：微信小程序与 Web 前台共用同一份 Taro 4 + Vue 3 源码，一码多端编译，业务逻辑只维护一份。小程序主攻微信生态社交裂变与支付闭环，Web 前台承接浏览器搜索流量、朋友圈/公众号 H5 落地、PC 端企业客户及品牌官网展示。

### 4.2 技术选型建议（MVP 阶段）
| 层 | 选型 | 理由 |
|---|---|---|
| C 端（小程序 + Web 前台） | Taro 4 + Vue 3 | 一码多端：同源编译微信小程序 + H5/PC Web，开发与维护成本最低 |
| Web 后台 | Vue 3 + Element Plus + Vite | 生态成熟，招聘容易 |
| 后端 | Node.js (NestJS) 或 Java (Spring Boot) | 看团队；MVP 期推荐 Node |
| 数据库 | MySQL 8 + Redis 7 | 业务/缓存分离 |
| 消息队列 | RabbitMQ 或阿里云 RocketMQ | 处理指令工单 |
| 对象存储 | 阿里云 OSS / 腾讯云 COS | 存储摄像头截图、短视频 |
| 推流/直播 | 腾讯云 LVB + 海康 ISUP 协议 | 摄像头接入成本低 |
| CDN | 阿里云/腾讯云 CDN | 视频回放加速 |
| 支付 | 微信支付（主）、支付宝（次） | 小程序天然打通 |
| IoT | 海康威视 Ehome 协议 / 萤石云 OpenAPI | 成熟方案，不自研 |

### 4.3 关键技术点
1. **摄像头接入**：采用萤石云 OpenAPI 或海康 ISUP，取得直播地址后由小程序 `<live-player>` 组件播放。
2. **指令执行闭环**：用户下发指令 → 服务端生成工单 → 农技员 Web端接收 → 执行后上传照片 → 用户收到推送。
3. **消息推送**：微信订阅消息（服务通知、生长节点、收获预告）。
4. **冷链物流**：对接顺丰冷运或京东物流 API，订单号回传用户。
5. **数据看板**：BI 选用开源 Metabase 或阿里云 QuickBI。
