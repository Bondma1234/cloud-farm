## 0. 文档说明

### 0.1 目的
将《云上田园项目商业计划书》中的业务构想转译为**可排期、可开发、可测试、可验收**的软件需求，为 MVP 版本落地提供唯一参考。

### 0.2 范围
- 包含：C 端微信小程序、C 端 Web 前台（H5/PC）、Web 管理后台、基础物联网对接、订单与支付主流程
- 不包含：自研硬件、自研 SaaS 多租户架构（v3 再议）、海外版

### 0.3 约定
- 优先级：**P0**（MVP 必须）、**P1**（v1 应有）、**P2**（v2 可有）
- 所有金额单位：元人民币
- 所有时间字段：服务端存储 UTC，展示为东八区

## 1. 产品总览

### 1.1 产品矩阵
| 端 | 名称 | 使用者 | 主要用途 |
|---|---|---|---|
| C 端·小程序 | 云上田园 微信小程序 | 认养用户、企业员工（移动端主入口） | 认养、互动、下单、观看直播、社交分享 |
| C 端·Web 前台 | 云上田园 官网/H5（浏览器） | 搜索引流用户、PC 端企业客户、品牌展示 | 认养、互动、下单、观看直播、SEO/朋友圈落地 |
| B 端 | 云上田园 运营后台（Web） | 平台运营、农场场长、农技员、客服、财务 | 订单、地块、工单、内容、财务、数据 |
| IoT | 田间摄像头/传感器 | 平台（系统自动） | 直播、数据上报 |

> C 端双端共享同一份 Taro 4 + Vue 3 源码（一码多端编译），功能范围与交互一致；仅在布局适配（移动端 vs 桌面）与支付/分享通道（小程序 JSAPI vs H5 JSSDK）上按端差异化实现。

### 1.3 术语与实体
- **地块（Plot）**：农场内的最小销售单元（10–20 ㎡），有唯一编号。
- **认养订单（Adoption）**：用户购买一个地块年度使用权的订单。
- **指令工单（Command Ticket）**：用户发起的浇水/施肥等互动请求，由农技员执行。
- **生长节点（Growth Milestone）**：作物关键阶段（播种、发芽、开花、结果、收获）。
- **寄送单（Shipment）**：作物收获后的物流配送单。

## 3. 核心业务流程

### 3.1 认养主流程
```
浏览首页 → 选套餐 → 选地块 → 选作物 → 填收货地址 → 微信支付
   → 生成认养订单 → 绑定地块（状态=已认养）→ 推送「欢迎卡片」
   → 农场排期播种 → 生长节点推送 → 用户互动 → 收获 → 寄送 → 完结/续费
```

### 3.2 指令互动流程
```
用户在小程序选择地块 → 点击「浇水/施肥/除草」
   → 服务端校验冷却时间、当季限额 → 生成工单（状态=待执行）
   → 推送到农技员 Web 端
   → 农技员执行并上传 ≥ 1 张执行照片
   → 状态=已完成 → 订阅消息推送用户 → 用户查看回执并可点赞
```

### 3.3 直播电商下单流程
```
用户在直播间/商城浏览 → 加购 → 结算 → 支付
   → 订单进入「待发货」→ 仓库打包 → 物流揽收
   → 回传单号 → 用户 App 查询 → 确认收货 → 可评价
```

### 3.4 企业认购流程
```
企业 BD 或企业主自助询盘 → 签合同 → 创建企业账户
   → 分配地块（可多块）→ 生成员工邀请码/链接
   → 员工用微信扫码加入企业圈 → 观看直播/生长报告
   → 成熟后按员工地址批量寄送 → 年度 ESG 报告导出
```

## 4. C 端功能需求（小程序 + Web 前台）

> 本章需求**同时适用于微信小程序与 Web 前台**（共享 Taro 4 + Vue 3 源码），差异点单独在"端差异"小节中标注。

### 4.1 信息架构
```
小程序：   首页 Tab  |  农场 Tab  |  直播/商城 Tab  |  我的 Tab   （底部 TabBar）
Web 前台： 顶部导航：首页 / 认养套餐 / 我的田 / 直播 / 我的      （响应式；≤768px 折叠为抽屉菜单）
```

### 4.0 端差异对照
| 能力 | 小程序 | Web 前台 | 实现方式 |
|---|---|---|---|
| 支付 | 微信支付 JSAPI（免跳转） | 微信 H5 支付 / 扫码支付（PC） | 按端注入不同适配器 |
| 分享 | onShareAppMessage / 朋友圈卡片 | 链接复制 + 微信 JSSDK（在微信内打开时） | 运行时探测 |
| 登录 | wx.login 静默授权 | 手机号 + 验证码 / 微信扫码登录 | 双入口并存 |
| 订阅消息 | 微信订阅消息模板 | 短信 / 邮件 / 浏览器 Web Push | 后端按端选择通道 |
| 摄像头直播 | `<live-player>` 原生组件 | HLS / FLV.js | 组件层条件编译 |
| SEO | 不支持 | 首页/套餐详情静态化 + meta 优化 | Vite SSG 预渲染 |
| 适配 | 固定移动布局 | 响应式：≤480 移动 / 481-768 平板 / ≥769 桌面 | CSS 媒体查询 + 容器查询 |

### 4.3 重点页面详细需求

#### 4.3.1 M-03 认养套餐（P0）
**页面组成**
1. 套餐卡片：基础版 / 进阶版 / 亲子版 / 企业版（企业版跳客服页）
2. 套餐对比表
3. "选地块" CTA

**功能点**
- 套餐详情页展示：含收获物清单、服务细则、退订政策、图文示例
- 地块选择器：
  - 以缩略地图 + 网格展示，按"大棚/区域"分组
  - 地块状态：`可认养` / `已认养` / `系统预留`（不可选）
  - 点击地块进入"地块预览"，展示编号、朝向、往年种植数据
- 作物选择：根据当季可种列表勾选 1–2 种
- 下单：唤起微信支付（JSAPI），支持使用优惠券/积分抵扣 ≤ 订单金额 10%

**异常**
- 库存并发：使用 Redis 预扣 10 分钟
- 支付中断：未支付单 30 分钟自动释放地块

**字段**
- 订单号、用户ID、地块ID、套餐ID、作物ID[]、金额、优惠券ID、积分抵扣、支付状态、时间戳

#### 4.3.2 M-04 地块详情（P0）
**页面组成**
1. 顶部：摄像头直播区（`<live-player>`）
2. 中部：地块信息卡（编号、面积、作物、种植日期、预计收获日）
3. 互动区：浇水/施肥/除草/捉虫/拍照 5 颗按钮
4. 下部：生长时间轴（最近 20 条动态）
5. 浮标：分享、命名、收获倒计时

**摄像头**
- 进阶版：支持云台控制（上下左右 + 变焦），指令防抖 500ms，同一时刻仅一用户可控
- 基础版：共享轮播摄像头，每 10 分钟切换
- 断流兜底：显示"最近一次快照"与"网络异常"提示

**命名功能**
- 一年内可修改 3 次，亲子版立牌仅首次免费，后续 20 元/次

#### 4.3.3 M-05 互动指令（P0）
**规则**
| 指令 | 冷却 | 季度上限 | 是否付费 |
|---|---|---|---|
| 浇水 | 6 小时 | 无 | 免费 |
| 施肥 | 7 天 | 6 次 | 免费 |
| 除草 | 3 天 | 12 次 | 免费 |
| 捉虫 | 按需 | 6 次 | 免费 |
| 拍照特写 | 2 天 | 20 次 | 免费 |
| 追加菜苗 | – | 2 次 | 付费（29 元起） |

**交互**
- 点击按钮 → 弹出确认 → 提交工单 → 页面展示"已排队，预计 X 小时内完成"
- 完成后订阅消息 + App 内红点 + 时间轴更新照片

**反作弊**
- 同 IP 同账户刷指令：1 分钟 3 次即进入风控观察
- 工单需地块归属校验

#### 4.3.4 M-07 直播（P0）
**直播间功能**
- 视频播放（`<live-player>` 拉流）
- 商品弹窗（下方滚动 3 个 SKU）
- 互动：点赞（本地计数 + 批量上报）、评论（敏感词过滤）
- 关注、分享
- 主播说明区："这里是您张先生家的田"——直播可"@"认养用户（需用户勾选允许）

**延迟要求**：端到端 ≤ 6 秒

#### 4.3.5 M-08 商城 / M-09 订单（P0）
**商品**
- 类目：鲜食、深加工、套餐箱、周边
- SKU：支持多规格（重量 × 包装）
- 库存：按日滚动，来自后台仓储模块

**购物车**
- 最多 20 个 SKU
- 支持满减、满赠、首单优惠

**结算**
- 可叠加 1 张优惠券 + 积分（上限 10%）+ 运费
- 冷链标的物自动加 20 元起运费，满 199 包冷链

**售后**
- 退款：未发货可随时退
- 退货退款：需上传图片、反馈原因、客服审核
- 补偿：作物绝收时，系统自动生成等价补偿订单供用户确认

#### 4.3.6 M-11 消息中心（P0）
**消息类型**
1. 服务通知（订阅消息模板）
   - 支付成功、发货、生长节点、工单完成、收获提醒、续费到期
2. 营销通知（每周上限 2 条，需用户勾选授权）
3. 系统公告

**模板 ID 规划**
- MSG_ADOPT_SUCCESS
- MSG_GROWTH_MILESTONE
- MSG_COMMAND_DONE
- MSG_SHIPMENT_OUT
- MSG_RENEWAL_REMIND

### 4.4 小程序非功能要点
- 包体积 ≤ 2 MB（首包），分包加载直播、商城
- 冷启动 ≤ 3 秒
- 兼容微信基础库 ≥ 2.19.0
- 深色模式 v1 不做

### 5.2 关键模块详细需求

#### 5.2.1 工作台（P0）
- 今日指标：新增用户、订单数、GMV、工单待处理数、摄像头离线数
- 待办：待执行工单、待发货订单、待退款单
- 近 30 天趋势图：GMV、新增认养、续费率
- 快捷入口

#### 5.2.2 认养订单（P0）
- 列表字段：订单号、用户、套餐、地块、金额、支付状态、创建时间、收获状态
- 筛选：状态、时间、套餐、地块基地、关键字
- 操作：详情、退款、改期、补发、转让（土地流转）、导出 Excel
- 批量：批量发货、批量打标签

#### 5.2.3 地块管理（P0）
- 地图式展示：基于平面图 SVG，点击地块见详情
- 属性：编号、基地、面积、朝向、状态（可售/已售/维护中/报废）
- 作物历史：按年份显示种植记录
- 批量导入：Excel 模板

#### 5.2.4 指令工单（P0）
- 工单列表，默认按紧急度排序
- 执行页（支持手机端）：拍照、备注、标记完成
- SLA 统计：平均响应时长、超时单
- 异常：48 小时未响应 → 自动通知场长 → 72 小时未处理 → 用户补偿券

#### 5.2.5 作物库 & 种植计划（P0）
- 作物库：名称、生长周期、最佳种植月、保底产量、图片、标签（亲子友好/耐储运）
- 种植计划：按地块 × 季度 × 作物 生成甘特图
- 预警：逾期未种、逾期未收

#### 5.2.6 摄像头 / IoT（P0）
- 设备列表：SN、型号、SIM、绑定地块、在线状态、最近心跳、信号强度
- 视频预览（嵌入播放器）
- 云台控制测试
- 告警：离线 > 30 分钟、信号 < 20%、SIM 流量 > 80%
- 对接萤石云 OpenAPI：设备添加、令牌、直播地址获取

#### 5.2.7 直播频道（P0）
- 频道：抖音、视频号、小程序直播间
- 排期日历
- 话术模板、商品弹窗脚本
- 直播复盘：观看数、点赞、引流到小程序 UV、下单转化率

#### 5.2.8 优惠券 / 老带新 / 营销（P1）
- 券模板：满减、折扣、兑换、新人券
- 发放渠道：手动、活动、裂变
- 老带新：邀请人奖励、被邀请人奖励、双方到账条件
- 节点活动：活动创建器（时间、素材、目标用户标签）

#### 5.2.9 物流 & 冷链（P0）
- 渠道：顺丰冷运、京东物流，API 接单/回传
- 发货单：按批次合并打印面单
- 退货：退货点对接
- 冷链温控（v2 可选）

#### 5.2.10 财务（P0）
- 日流水：微信支付、支付宝
- 对账：平台 vs. 微信账单 差异报告
- 发票：用户申请 → 财务审核 → 对接税务平台
- 提现：给农场/场长的分成

#### 5.2.11 数据看板（P1）
- 业务大盘：GMV、认养数、续费率、LTV
- 漏斗：UV → 详情页 → 下单 → 支付
- 渠道：抖音、视频号、小红书带来的注册 & 转化
- 地块 ROI 排行

#### 5.2.12 企业账户（P1）
- 企业主页：公司名、税号、合同、开票信息
- 员工子账户批量导入 & 邀请码
- 企业可见的员工观看报表
- 年度 ESG 报告 PDF 导出

#### 5.2.13 系统设置 / 权限（P0）
- RBAC：用户 - 角色 - 权限
- 操作日志：6 个月可查
- 基础配置：支付参数、短信模板、微信模板 ID、物流密钥（加密存储）

## 6. 物联网 / 硬件接入需求

### 6.1 摄像头
- 型号建议：萤石 C6 太阳能版 或 海康 4G 球机
- 接入方式：萤石云 OpenAPI（GET token → 直播地址）
- 规格：1080p、夜视、云台（进阶版）、4G 双卡、IP66
- 每个基地部署一台 4G 聚合路由作为兜底

### 6.2 传感器（P1）
- 土壤温湿度、空气温湿度、光照度
- 5 亩 1 组，LoRa 上报网关 → MQTT → 平台
- 展示：地块详情图表；低于阈值自动生成养护建议工单

### 6.3 硬件 SLA
- 在线率 ≥ 99%
- 故障响应：远程 2 小时、到场 48 小时

## 7. 接口与数据规范

### 7.1 接口规范
- 协议：HTTPS + JSON
- 鉴权：JWT（C 端）、Session+RBAC（B 端）
- 版本：URL 前缀 `/api/v1`
- 错误码规范：`{code: 0, msg, data}`；`code != 0` 为业务错误
- 分页：`page`、`size`，最大 100/页

### 7.2 核心实体（节选）
- `user(id, openid, unionid, phone, nickname, avatar, role, created_at)`
- `plot(id, base_id, code, size, status, location_svg_id)`
- `adoption(id, user_id, plot_id, package_id, crops[], amount, status, start_at, end_at)`
- `command(id, user_id, plot_id, type, status, cooldown_until, created_at, finished_at, images[])`
- `growth_log(id, plot_id, milestone, image_urls[], note, logged_by, logged_at)`
- `order(id, user_id, type, items[], amount, payment_status, shipping_status)`
- `shipment(id, order_id, carrier, tracking_no, status, timeline[])`
- `device(id, sn, model, plot_id, online, signal, last_heartbeat_at)`

### 7.3 关键接口清单（节选，P0）
| 模块 | 方法 | 路径 |
|---|---|---|
| 登录 | POST | /api/v1/auth/wx-login |
| 套餐列表 | GET | /api/v1/packages |
| 可选地块 | GET | /api/v1/plots?status=available&base_id=… |
| 创建认养订单 | POST | /api/v1/adoptions |
| 地块详情 | GET | /api/v1/plots/{id} |
| 发起指令 | POST | /api/v1/commands |
| 生长日志 | GET | /api/v1/plots/{id}/growth |
| 直播列表 | GET | /api/v1/lives |
| 商品/下单 | CRUD | /api/v1/goods, /api/v1/shop-orders |
| 消息订阅 | POST | /api/v1/subscribe |
| 物流查询 | GET | /api/v1/shipments/{id} |

### 7.4 数据合规
- 个人信息最小化收集
- 摄像头直播视频不落本地磁盘，仅 CDN 存 72 小时回放
- 用户注销 → 30 天内清理个人数据
- 遵循《个人信息保护法》《数据安全法》
