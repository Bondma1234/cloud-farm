# 云上田园 · 产品需求说明书（PRD）v2.0

> 文档版本：v2.0
> 更新日期：2026-04-28
> 修订人：binbin
> 适用范围：C 端微信小程序 + C 端 Web Portal + B 端 Web 后台 + 摄像头硬件接入
>
> **本版相对 v1 的核心变化（与架构 v2 同步）：**
>
> 1. **彻底删除"直播"概念** —— 我们做的是 1 摄像头 → 1 用户的**私有远程监控**（用户在小程序/网页看自己认养地块的摄像头实时画面），**不是直播**。1 推流 → N 观众的直播架构、观看人数、弹幕、礼物、直播间下单等全部移除
> 2. **C 端拆分** —— miniapp（Taro）保留作为微信小程序，新增 web（独立 Vue 3 + Vite）作为 C 端 Web Portal，服务"喜欢电脑浏览的用户"和"大屏看监控更清晰"的场景
> 3. **底部导航重排** —— 删除原"直播 tab"，C 端导航简化为：首页 / 认养 / 我的田 / 我的（4 个）
> 4. **后台"直播频道"模块改名"外部营销渠道"** —— 抖音/视频号直播带货是项目书原本的营销手段，但**那是抖音平台的事**，我们后台只做对接（话术模板、复盘数据），不做推流
> 5. **摄像头模块升级到核心功能** —— 从原"M-04 地块详情中的一个组件"扩展为独立模块 M-07，含 EZOPEN 取流、PTZ 云台、拍照抓帧、播放鉴权
>
> v1 文档保留作历史参考，**新决策一律以本 v2 为准**。

## 0. 文档说明

### 0.1 目的
将《云上田园项目商业计划书》中的业务构想转译为**可排期、可开发、可测试、可验收**的软件需求，为 MVP 版本落地提供唯一参考。

- 不包含：自研硬件、自研 SaaS 多租户架构（v3 再议）、海外版、**自有直播业务**

- 所有金额单位：元人民币
- 所有时间字段：服务端存储 UTC，展示为东八区

## 1. 产品总览

### 1.1 产品矩阵
| 端 | 名称 | 使用者 | 主要用途 |
|---|---|---|---|
| C 端·小程序 | 云上田园 微信小程序 | 认养用户、企业员工（移动端主入口） | 认养、互动、下单、**查看自己摄像头**、社交分享 |
| C 端·Web Portal | 云上田园 官网 cloud-farm.com（浏览器） | 喜欢电脑浏览的用户、PC 端企业客户、SEO 引流、品牌展示 | 认养、互动、下单、**电脑大屏看摄像头**、SEO/分享落地 |
| B 端 | 云上田园 运营后台（Web） | 平台运营、农场场长、农技员、客服、财务 | 订单、地块、工单、动态、财务、数据 |
| IoT | 田间摄像头 | 平台（系统接入萤石云） | 用户私有监控、农技员协助巡田 |

> **架构 v2 关键变更**：C 端不再"一份 Taro 代码两端编译"，而是**两个独立项目**（miniapp + web）共享 packages/shared 数据层与 ui-tokens 设计层。详见 [架构 v2 §2](./03_云上田园_软件架构图_v2.md)。

### 1.2 系统边界图
```
[用户] ──┐                                ┌──► [微信支付]
         │                                │
         ▼   (小程序 / Web Portal / 后台)  ▼
      [前端] ◄─── HTTPS ─────────────► [NestJS API]
                                         │
         ┌────────────────┬──────────────┼──────────────────┐
         ▼                ▼              ▼                  ▼
     [MySQL]          [Redis]         [OSS]            [萤石云 API]
                                                         │
                                                         ▼
                                                [海康/萤石 摄像头]
                                                         │

```

**与 v1 边界图对比**：
- ❌ 删除 "直播 CDN"（不存在直播）
- ❌ 删除 "IoT 网关 / MQTT"（萤石云 SaaS 包办）
- ✅ 新增 "萤石云 API"（摄像头取流的唯一入口）

### 1.3 术语与实体
- **地块（Plot）**：农场内的最小销售单元（10–20 ㎡），有唯一编号
- **认养订单（Adoption）**：用户购买一个地块年度使用权的订单
- **指令工单（Command Ticket）**：用户发起的浇水/施肥/拍照等请求，由农技员执行
- **生长节点（Growth Milestone）**：作物关键阶段（播种、发芽、开花、结果、收获）
- **寄送单（Shipment）**：作物收获后的物流配送单
- **摄像头（Camera）** ★新：绑定到地块的海康/萤石设备，1 个地块对应 1 个摄像头
- **抓拍照片（Snapshot）** ★新：用户点"拍张照"指令时，后端从摄像头拉一帧存 OSS 的图片

## 3. 核心业务流程

### 3.1 认养主流程
```
浏览首页 → 选套餐 → 选地块 → 选作物 → 填收货地址 → 微信支付
   → 生成认养订单 → 绑定地块（状态=已认养）→ 自动绑定摄像头
   → 推送「欢迎卡片」→ 农场排期播种 → 生长节点推送
   → 用户互动（包含每天看摄像头）→ 收获 → 寄送 → 完结/续费
```

### 3.2 指令互动流程
```
用户在小程序选择地块 → 点击「浇水/施肥/除草/捉虫/拍张照」
   → 服务端校验冷却时间、当季限额 → 生成工单（状态=待执行）
   → 推送到农技员 Web 端
   → 农技员执行并上传 ≥ 1 张执行照片 (拍张照指令由后端自动从摄像头抓帧)
   → 状态=已完成 → 订阅消息推送用户 → 用户查看回执并可点赞
```

### 3.3 农产品商城下单流程 ★ 替代 v1 的"直播电商下单流程"
```
用户在小程序/Web Portal 浏览商城 → 加购 → 结算 → 微信支付
   → 订单进入「待发货」→ 仓库打包 → 物流揽收
   → 回传单号 → 用户 App 查询 → 确认收货 → 可评价
```

> **v1 假设的"直播间下单"链路移除**。如果运营在抖音/视频号直播带货引流到我们小程序下单，那是抖音平台的事，与我们的应用架构无关。

### 3.4 摄像头查看流程 ★ 新增
```
用户进"我的田" → 前端请求 GET /api/cameras/:plotId/url
   → 后端鉴权（plotId 是否属于当前用户）→ Redis 看是否有缓存的播放 url
      ├─ 有：直接返回（缓存 8 分钟）
      └─ 无：调萤石云 OpenAPI 取临时播放地址（EZOPEN 协议）→ 缓存到 Redis → 返回
   → 前端用 <live-player>(小程序) 或 flv.js (Web) 播放
   → 用户可点击 PTZ 按钮控制云台（仅进阶版/亲子版） → 后端转发到萤石 API
   → 用户可点"拍张照"（走 §3.2 指令流程）
```

### 3.5 企业认购流程
```
企业 BD 或企业主自助询盘 → 签合同 → 创建企业账户
   → 分配地块（可多块）→ 自动绑定对应摄像头
   → 生成员工邀请码/链接 → 员工用微信扫码加入企业圈
   → 员工可查看本企业的摄像头与生长报告（不能下单）
   → 成熟后按员工地址批量寄送 → 年度 ESG 报告导出
```

## 4. C 端功能需求（小程序 + Web Portal）

> 本章需求**同时适用于微信小程序与 Web Portal**。架构 v2 后这是两个独立项目，但**业务功能、视觉风格、交互逻辑保持一致**，仅在技术实现上按端差异化（详见 §4.0 端差异）。

### 4.1 信息架构

```
小程序底部 TabBar：  首页  |  认养  |  我的田  |  我的       （4 个 tab）

Web Portal 顶部导航： 首页 / 认养套餐 / 我的田 / 我的账户   （响应式；
                     ≤768px 折叠为抽屉菜单 + 底部 4-tab 模拟移动布局）
```

**v1 → v2 变更**：删除原"直播/商城" tab，商城功能并入"首页"中的"农产品"入口与"我的"中的订单。"我的田"内置摄像头查看（核心功能），不再独立成 tab。

### 4.0 端差异对照
| 能力 | 小程序 | Web Portal | 实现方式 |
|---|---|---|---|
| 框架 | Taro 4 + Vue 3 | Vue 3 + Vite + vue-router(history) | 两个独立项目 |
| 支付 | 微信支付 JSAPI | 微信 H5 支付 / 微信扫码（PC） | 后端按端选 prepay 模式 |
| 分享 | onShareAppMessage / 朋友圈卡片 | URL 复制 + 微信 JSSDK（在微信内打开时） | 各端原生实现 |
| 登录 | wx.login 静默 | 短信验证码 / 微信扫码登录 | 双入口并存 |
| 订阅消息 | 微信订阅消息模板 | 短信 + 浏览器 Web Push（可选） | 后端按端选通道 |
| **摄像头实时画面** | `<live-player>` + 萤石 EZOPEN | `<video>` + flv.js / 萤石 H5 SDK | 各端原生 |
| SEO | 不支持（小程序无搜索引擎） | 套餐详情页 SSR/SSG，sitemap.xml | Web 独享 |
| 适配 | 移动布局 | 响应式：移动 / 平板 / **桌面大屏（看摄像头清晰）** | CSS 媒体查询 |

**v1 → v2 变更**：
- 框架行：从"共享 Taro 一码多端"改为"两个独立项目"
- ~~"摄像头直播"行~~ 改名为 **"摄像头实时画面"**
- 适配行强调"桌面大屏看摄像头清晰" —— 这是 Web Portal 的关键差异化卖点

### 4.3 重点页面详细需求

#### 4.3.1 M-03 认养套餐（P0）
**页面组成**
1. 套餐卡片：基础版 / 进阶版 / 亲子版 / 企业版（企业版跳客服页）
2. 套餐对比表
3. "选地块" CTA

**功能点**
- 套餐详情页展示：含收获物清单、服务细则、退订政策、图文示例
- 地块选择器：
  - 缩略地图 + 网格展示，按"大棚/区域"分组
  - 地块状态：`可认养` / `已认养` / `系统预留`（不可选）
  - 点击地块进入"地块预览"，展示编号、朝向、往年种植数据、**绑定的摄像头位置**
- 作物选择：根据当季可种列表勾选 1–2 种
- 下单：唤起微信支付（JSAPI/H5），支持优惠券/积分抵扣 ≤ 订单金额 10%

**异常**
- 库存并发：使用 Redis 预扣 10 分钟
- 支付中断：未支付单 30 分钟自动释放地块

#### 4.3.2 M-04 我的田（P0）
**页面组成**
1. 顶部状态卡：地块编号 + 作物 emoji + 生长阶段 + 进度条 + 下一步动作 + 当前天气
2. **摄像头实时画面**（M-07 模块嵌入）—— PTZ 控制覆盖在画面上 + "观看完整大图"按钮
3. 互动按钮区：浇水/施肥/除草/拍张照（4 颗大按钮）+ "指令历史 ›"链接
4. 生长日志时间轴（最近 6 条）+ "查看全部 ›"链接到 M-06
5. 收获预估卡：剩余天数 / 预估产量 / 已用指令次数

**v1 → v2 变更**：把 v1 "顶部摄像头直播区"改名"实时画面"；摄像头是 M-07 独立模块的一个"嵌入位"。

#### 4.3.3 M-05 互动指令（P0）
**规则**
| 指令 | 冷却 | 季度上限 | 是否付费 |
|---|---|---|---|
| 浇水 | 6 小时 | 无 | 免费 |
| 施肥 | 7 天 | 6 次 | 免费 |
| 除草 | 3 天 | 12 次 | 免费 |
| 捉虫 | 按需 | 6 次 | 免费 |
| **拍张照（从摄像头抓帧）** | 8 小时 | 30 次 | 免费 |
| 追加菜苗 | – | 2 次 | 付费（29 元起） |

**交互**
- 点击按钮 → 弹出确认 → 提交工单 → 页面展示"已排队，预计 X 小时内完成"
- 完成后订阅消息 + 红点 + 时间轴更新照片
- "拍张照"指令由后端自动从摄像头抓帧（v1 是"农技员人工拍"，v2 改为"系统自动 + 农技员人工备用"）

#### 4.3.4 M-07 摄像头远程查看（P0）★ v2 新增重点

**核心交互**
1. **取流**：进入页面 → 自动请求播放 url（带鉴权）→ 播放器自动 attach
2. **PTZ 云台**（仅进阶版/亲子版）：
   - 上、下、左、右、放大、缩小 6 个按钮
   - 按下后端 200ms 内响应（萤石 API 转发）
   - 同一摄像头同一时刻仅 1 用户可控（基础版套餐说明里写明"共享摄像头不可控制"）
3. **拍照**：触发 M-05 "拍张照"指令，2 秒内画面上闪一下并提示"照片已存到生长日记"
4. **横屏 / 全屏**（Web Portal 专属）：电脑大屏点击全屏按钮，画面铺满显示器
5. **画质切换**（v1）：流畅 480p / 高清 720p / 超清 1080p
6. **断线兜底**：
   - 摄像头离线 → 显示"最近一次抓拍图"+"摄像头维护中"+客服联系入口
   - 网络问题 → 自动重试 3 次，再失败提示"刷新重试"

**鉴权规则**
- 必须登录
- 必须是该地块的认养人 / 企业员工 / 农技员（场长可看所有）
- 1 用户 1 分钟最多 10 次取流请求（防爆刷萤石 API）

**画面延迟要求**
- 端到端 ≤ 30 秒（场景是看长势，不是直播）
- 萤石云 EZOPEN 协议本身延迟约 5-15 秒

**v2 关键决策**：摄像头**不**支持评论 / 弹幕 / 点赞 / 商品弹窗等所有直播间常见功能。这是私有空间，不是公开直播。

#### 4.3.5 M-08 商城 / M-09 订单（P0）
**商品**
- 类目：鲜食、深加工、套餐箱、周边
- SKU：支持多规格（重量 × 包装）
- 库存：按日滚动，来自后台仓储模块

**购物车**
- 最多 20 个 SKU
- 支持满减、满赠、首单优惠

**结算**
- 可叠加 1 张优惠券 + 积分（上限 10%）+ 运费
- 冷链标的物自动加 20 元起运费，满 199 包冷链

**售后**
- 退款：未发货可随时退
- 退货退款：需上传图片、反馈原因、客服审核
- 补偿：作物绝收时，系统自动生成等价补偿订单供用户确认

#### 4.3.6 M-11 消息中心（P0）
**消息类型**
1. 服务通知（订阅消息模板）

4. **摄像头离线告警**（仅推一次/天，避免轰炸）★

### 4.4 小程序非功能要点

## 6. 摄像头与 IoT 接入需求

### 6.1 摄像头硬件
- **型号建议**：萤石 C6 太阳能版（户外）或 海康 4G 球机
- **接入方式**：萤石云 OpenAPI（GET token → EZOPEN 播放地址）
- **规格**：1080p、夜视、云台（进阶版/亲子版套餐）、4G 双卡、IP66
- 每个基地部署一台 4G 聚合路由作为兜底网络

### 6.2 摄像头数据流
```
摄像头 ─RTSP─▶ 萤石云内部 ─转码─▶ EZOPEN/HLS/FLV
                                  │
                                  ▼ HTTPS（带签名 token，10min 有效）
                          [我们的 API]
                                  │
                                  ▼ token 转发
                          [小程序 <live-player> / Web <video> + flv.js]
```

### 6.3 PTZ 云台控制
- 6 个方向 + 变焦 + 重置位
- 后端转发萤石 OpenAPI，200ms 内响应
- 同一摄像头同一时刻仅 1 用户控制（用 Redis lock 实现）

### 6.4 抓拍（拍张照指令）
- 后端调萤石 OpenAPI 拉一帧 jpeg → 上传 OSS → 写 Command + JournalEntry → 推用户消息

### 6.5 SLA
- 摄像头在线率 ≥ 95%（户外 4G，达不到 99% 是合理的）
- 故障响应：远程 2 小时定位、到场 48 小时
- **不要求超低延迟**（≤ 30 秒画面延迟可接受）

### 6.6 ~~传感器~~（v2 移除，MVP 不要）
v1 提到的土壤温湿度、空气温湿度、光照度传感器：**MVP 不接**。理由：传感器布设成本高、农场实际管理还是看人 + 摄像头画面就够，等业务跑起来 v1 阶段再说。

- 所有 P0 功能覆盖单元测试（后端 ≥ 60%）与 E2E 主流程
- Bug 等级：Blocker 0 / Critical 0 / Major ≤ 3

### 11.2 主流程用例（节选）
- UC-01：新用户完成认养下单（包含优惠券使用）
- UC-02：用户发起浇水指令 → 农技员完成 → 推送回用户
- UC-03：**用户进我的田 → 看到摄像头实时画面 → PTZ 转一下** ★ 替代 v1 的"直播间下单"
- UC-04：**用户点拍张照 → 后端抓帧 → 自动加入生长日记** ★
- UC-05：**摄像头离线 → 后台告警 → 场长处理 → 用户消息推送** ★ 调整
- UC-06：作物绝收场景 → 后台批量生成补偿券
- UC-07：**Web Portal 桌面浏览器全屏看摄像头 ≥ 5 分钟流畅** ★
- UC-08：订阅消息下发 → 用户 24h 内打开复率 ≥ 30%

### 11.3 性能/安全测试
- 压测：认养下单 200 并发；**摄像头取流接口 100 并发**（不再压 1000 直播在线）
- 渗透测试：OWASP Top 10
- **摄像头授权测试**：A 用户尝试取 B 用户摄像头，必须 403
- 隐私测试：小程序权限弹窗符合微信规范

## 附录 A：角色-故事（User Story）示例（v2 调整）

- **As a** 北京宝妈（核心用户 A），**I want to** 在小程序里看孩子认养地块的摄像头画面 + 每天的长势照片，**so that** 给孩子讲"食物是怎么来的"。
- **As a** 上海上班族（核心用户 B）★ 新，**I want to** 在公司用电脑大屏看自己的农场摄像头，**so that** 累的时候放松一下。
- **As a** 认养用户，**I want to** 给地块命名并看到立牌照片，**so that** 增加归属感。
- **As a** 农技员，**I want to** 在手机上一键完成工单并上传照片，**so that** 不用在办公室电脑前操作。
- **As a** 企业行政，**I want to** 导出 ESG 年度报告 PDF，**so that** 汇报管理层。
- **As a** 运营，**I want to** 监控摄像头在线率 + 抖音直播带来的引流转化，**so that** 在用户投诉之前先处理 / 评估渠道效果。

## 附录 B：v2 相比 v1 的变更总览

| 维度 | v1 | v2 |
